# README – Kit de Encuesta sobre Indicadores y Métricas ISO/IEC 27001:2022

Bienvenido/a al Kit de Encuesta sobre Indicadores y Métricas ISO/IEC 27001:2022. Este README explica qué contiene el kit, cómo utilizarlo y cómo no perder la cordura en el proceso.

## 1. Contenido del kit

El kit está compuesto por los siguientes archivos:

1. `01_encuesta_iso27001.md`  
   Modelo de cuestionario integral para responsables de seguridad, riesgos, continuidad, etc. Incluye preguntas cerradas y espacios abiertos.

2. `02_guia_metodologica_encuesta.md`  
   Explica los fundamentos normativos, objetivos, población objetivo, metodología de aplicación y criterios de análisis.

3. `03_narrativa_explicativa.md`  
   Relato interpretativo sobre el sentido de la encuesta, orientado a facilitar su comprensión y aceptación por parte de la organización.

4. `04_mapeo_preguntas_norma.md`  
   Tabla de mapeo entre bloques de preguntas y requisitos de ISO/IEC 27001:2022, ISO/IEC 27004, guías de ISACA y marcos regulatorios.

5. `05_plantilla_excel_igm_roi.md`  
   Descripción detallada de una estructura de Excel para cálculo del Índice Global de Madurez (IGM) y ROI de seguridad.

6. `06_plantilla_reporte_ejecutivo_ppt.md`  
   Guion estructurado para elaborar un reporte ejecutivo tipo PowerPoint a partir de los resultados de la encuesta.

7. `07_README_kit_encuesta.md`  
   Este mismo README.

## 2. Cómo empezar

1. **Definir el alcance**  
   Decida a qué organizaciones se aplicará la encuesta (por ejemplo, un sector, un territorio, un grupo empresarial). Documente este alcance para contextualizar después los resultados.

2. **Adaptar mínimamente la encuesta**  
   Revise el archivo `01_encuesta_iso27001.md` y, si lo desea, añada o ajuste bloques para casos específicos (por ejemplo, OT, DORA, requisitos nacionales adicionales). Mantenga, en la medida de lo posible, la estructura básica para preservar comparabilidad. [web:13][web:33]

3. **Construir el formulario**  
   Transfiera las preguntas a la herramienta de encuesta elegida (formularios corporativos, plataformas especializadas, etc.). Asegúrese de que las opciones de respuesta se mantienen fieles al modelo.

4. **Comunicar el propósito**  
   Use la `03_narrativa_explicativa.md` como base para explicar a los participantes qué se pretende, qué no se pretende y por qué sus respuestas importan.

## 3. Recogida y tratamiento de datos

- Asegure confidencialidad y, si procede, anonimización de los resultados.
- Establezca un calendario razonable de respuesta.
- Prepare el libro de Excel según `05_plantilla_excel_igm_roi.md` para poder cargar y analizar las respuestas. [web:4][web:13]

Es recomendable que al menos una persona del equipo tenga experiencia básica en análisis de datos para facilitar el tratamiento y la producción de gráficos.

## 4. Análisis y reporte

1. **Cálculo de índices**  
   Utilice la hoja de cálculo para calcular el IGM y subíndices temáticos, así como los indicadores económicos si dispone de datos suficientes.

2. **Interpretación**  
   Apóyese en la `02_guia_metodologica_encuesta.md` y `03_narrativa_explicativa.md` para interpretar resultados: dónde hay fortalezas, dónde hay huecos, qué tendencias aparecen.

3. **Reporte ejecutivo**  
   Construya la presentación siguiendo `06_plantilla_reporte_ejecutivo_ppt.md`, adaptando gráficos y mensajes a la realidad observada.

## 5. Mantenimiento y evolución del kit

El mundo no se detiene en la versión 2022 de ISO 27001 ni en la transposición de NIS2. Los indicadores evolucionan, los organismos nacionales publican nuevos informes, y los auditores siempre descubren nuevas formas de hacer preguntas. [web:16][web:20][web:22]

Se recomienda:

- Revisar al menos cada 12–18 meses la encuesta y el mapeo normativo.
- Incorporar nuevos indicadores relevantes (por ejemplo, ligados a nuevas regulaciones).
- Ajustar el modelo de Excel para reflejar aprendizajes acumulados.

## 6. Condiciones de uso (cordura incluida)

Puede utilizar, adaptar y ampliar este kit dentro de su organización o consorcio. Se sugiere:

- Citar las normas y guías en las que se basa (ISO/IEC 27001:2022, ISO/IEC 27004, guías de ISACA, buenas prácticas recientes sobre KPIs de seguridad, informes de INCIBE y ENISA). [web:1][web:13][web:12][web:16]
- Documentar sus adaptaciones para que futuras iteraciones no se pierdan en la niebla.

## 7. Y ahora, ¿qué?

Una vez que tenga resultados:

- Úselos para priorizar acciones, no solo para escribir informes.
- Revise cuántos indicadores actuales pueden retirarse con elegancia.
- Identifique 5–10 indicadores realmente críticos para su organización, y asegúrese de que están bien definidos, bien medidos y bien utilizados.

Con eso, el SGSI no será perfecto, pero al menos sus indicadores dejarán de ser un ritual y empezarán a comportarse como herramientas de gestión de verdad.
