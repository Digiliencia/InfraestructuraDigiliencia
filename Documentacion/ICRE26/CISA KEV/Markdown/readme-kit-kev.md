# README del Kit de Encuesta KEV + GQM + PRAGMATIC

## 1. ¿Qué es este kit?

Este kit reúne los elementos necesarios para diagnosticar la madurez en:

- Gestión de vulnerabilidades con foco en **Known Exploited Vulnerabilities (KEV)**.
- Pruebas de penetración y ejercicios avanzados.
- Monitorización y gestión de eventos de seguridad (SIEM/SOC).
- Capacitación y cultura de seguridad.

Todo ello apoyado en:

- Metodología **GQM** (Goal–Question–Metric) para garantizar trazabilidad desde objetivos nacionales hasta métricas.
- Evaluación **PRAGMATIC** de las métricas para asegurar que lo que se mide es útil, accionable y sostenible.

## 2. Contenidos del kit

El kit (en formato Markdown) incluye:

1. **Modelo de encuesta integral** (encuesta KEV–vulnerabilidades–pentesting–SIEM–capacitación).
2. **Marco integrativo GQM+PRAGMATIC** (`marco-gqm-pragmatic.md`).
3. **Matriz de Evaluación PRAGMATIC Completa** (`matriz-pragmatic.md`).
4. **Mapeo normativo** (`mapeo-normas-kev.md`).
5. **Plantilla de Excel para IGM y ROI** (`plantilla-igm-roi-kev.md`).
6. **Plantilla de Reporte Ejecutivo** (`ppt-reporte-kev.md`).
7. Este **README** como guía de uso.

## 3. Cómo usar el kit (visión rápida)

1. **Definir el alcance del diagnóstico**  
   - ¿Organización única, grupo de entidades, sector, ámbito nacional?

2. **Adaptar el cuestionario**  
   - Revisar el modelo y ajustar redacción si es necesario, manteniendo los ID de preguntas.

3. **Implementar la encuesta**  
   - Cargar las preguntas en la herramienta de encuestas elegida.
   - Definir quién responde (CISO, TI, SOC, RRHH formación, etc.).

4. **Montar el Excel para el IGM y el ROI**  
   - Seguir la especificación de `plantilla-igm-roi-kev.md`.
   - Configurar hojas, rangos y fórmulas.

5. **Volcar y procesar los resultados**  
   - Importar las respuestas a la hoja `Respuestas`.
   - Calcular subíndices de madurez e IGM.
   - Evaluar iniciativas y ROI en la hoja `ROI`.

6. **Preparar el reporte ejecutivo**  
   - Seguir la estructura de `ppt-reporte-kev.md`.
   - Seleccionar los gráficos e indicadores más relevantes para la audiencia.

## 4. GQM + PRAGMATIC en dos frases

- **GQM**: empieza siempre por el objetivo (*Goal*), formula las preguntas clave (*Questions*) y solo entonces define qué métricas (*Metrics*) tienen sentido.
- **PRAGMATIC**: revisa cada métrica con nueve filtros (Predictivo, Relevante, Accionable, Genuino, Significativo, Preciso, Oportuno, Independiente y Rentable) para no caer en la trampa de medir por costumbre.

## 5. Buenas prácticas de despliegue

- **Transparencia**: explicar a los participantes el propósito del ejercicio: mejora, no «caza de brujas».
- **Confidencialidad**: proteger adecuadamente los datos de cada organización.
- **Iteración**: empezar con una primera oleada, aprender y ajustar para la siguiente.
- **Trazabilidad**: mantener alineados cuestionario, Excel, GQM y PRAGMATIC para que las métricas tengan siempre apellidos y propósito.

## 7. Y ahora, ¿qué?

Con el kit montado, el siguiente paso es casi existencial: **pasar de la intención a la medición**.  
Responder la encuesta, calcular el IGM y mirar de frente los indicadores KEV es mucho más sano que esperar a que sea un ransomware el que haga de auditor inesperado.

Si en algún punto del proceso siente una punzada de ironía al ver reflejada la realidad, enhorabuena: significa que el kit está funcionando.
