# README – Kit TRIKE + GQM + PRAGMATIC
## Versión 1.0

### 1. Contenido del kit

Este kit complementa el cuestionario TRIKE original incorporando un marco formal de métricas basado en GQM y evaluado con los criterios PRAGMATIC. [web:38][web:39][web:45]

Incluye:

- `a_marco_integrativo_gqm_pragmatic.md` – Descripción del marco GQM aplicado a TRIKE.
- `b_matriz_evaluacion_pragmatic.md` – Matriz de puntuación PRAGMATIC para las métricas clave.
- `c_narrativa_explicativa_gqm_pragmatic.md` – Narrativa de apoyo para responsables de política y dirección.
- `d_mapeo_normativo_gqm_pragmatic.md` – Mapeo detallado de métricas a NIS2, ENISA y normativa española.
- `e_plantilla_excel_igm_roi_gqm_pragmatic.md` – Especificación para hoja de cálculo con IGM y ROI integrados.
- `f_plantilla_reporte_ejecutivo_gqm_pragmatic.md` – Plantilla de reporte estratégico.

### 2. Uso recomendado

1. Definir, a nivel de país o sector, los objetivos de resiliencia y cumplimiento que se quieren medir.
2. Seleccionar el subconjunto de métricas TRIKE más alineado con esos objetivos, usando la matriz PRAGMATIC para priorizar. [web:45]
3. Implementar la plantilla de Excel y los procesos de recogida de datos (CSIRTs, reguladores, encuestas).
4. Generar informes periódicos (anuales o bienales) siguiendo la plantilla de reporte ejecutivo.

### 3. Personalización

- Las métricas propuestas pueden adaptarse por sector (sanidad, energía, administración, etc.).
- Los scores PRAGMATIC deben revisarse con expertos nacionales y sectoriales.
- El marco puede ampliarse con nuevas métricas emergentes (IA, OT, satélites, etc.). [web:46]

### 4. Limitaciones

Este kit ofrece un marco conceptual y técnico, no una base de datos ya poblada.  

El éxito depende de la calidad y la continuidad en la recogida de datos, así como de la voluntad política y organizativa de usar los resultados para algo más que rellenar informes. [web:16][web:41]
