# Kit GQM + PRAGMATIC para indicadores FAIR‑STRIDE – README

## 1. Contenido del kit

Este kit complementa la encuesta FAIR‑STRIDE incorporando el enfoque GQM + PRAGMATIC para el diseño y la gobernanza de indicadores:

- **Marco integrativo GQM + PRAGMATIC**: documento que explica cómo derivar métricas FAIR‑STRIDE desde objetivos nacionales, sectoriales u organizativos.
- **Matriz de Evaluación PRAGMATIC**: tabla con ejemplos de puntuación PRAGMATIC para las principales métricas propuestas.
- **Narrativa explicativa**: texto orientado a responsables de alto nivel que contextualiza el uso del marco.
- **Mapeo normativo**: tabla que vincula objetivos y métricas con requisitos de ISO 27001/27005, NIST CSF/RMF, NIS2 y DORA.
- **Plantilla de Excel**: guía para implementar el cálculo de IGM y ROI incorporando GQM y PRAGMATIC.
- **Plantilla de Reporte Ejecutivo**: estructura sugerida de presentación para comunicar resultados a niveles directivos y reguladores.

## 2. Público objetivo

El kit está pensado para:

- Organismos públicos nacionales y autonómicos responsables de estrategias de ciberseguridad y resiliencia digital.
- Reguladores sectoriales (finanzas, energía, sanidad, transporte) que necesitan indicadores comparables.
- Grandes organizaciones que deseen alinear su propia métrica FAIR‑STRIDE con objetivos nacionales.

## 3. Cómo usar este kit junto con la encuesta FAIR‑STRIDE

1. Utilice la **encuesta FAIR‑STRIDE** para obtener una visión de madurez organizativa.
2. A partir de los resultados, identifique los objetivos prioritarios (por ejemplo, mejorar disponibilidad sanitaria, reducir brechas de datos).
3. Use el **Marco GQM + PRAGMATIC** para derivar, desde esos objetivos, un conjunto de métricas FAIR‑STRIDE coherentes.
4. Evalúe las métricas candidatas con la **Matriz PRAGMATIC** y seleccione las más valiosas.
5. Implemente la **Plantilla de Excel** para calcular IGM, subíndices y ROI.
6. Presente los resultados mediante la **Plantilla de Reporte Ejecutivo**, ajustada a su contexto.

## 4. Recomendaciones de implantación

- Comience con un conjunto reducido de objetivos y métricas, preferiblemente en un sector o ámbito piloto.
- Documente claramente la trazabilidad GQM de cada métrica y haga transparente la puntuación PRAGMATIC.
- Revise periódicamente el set de métricas: elimine las que no aportan valor, refuerce las que sí.
- Evite vincular directamente métricas frágiles a incentivos fuertes sin antes evaluar su independencia.

## 5. Mantenimiento y evolución

El paisaje normativo y tecnológico cambia con rapidez. Se recomienda:

- Revisar el kit cada 1‑2 años para incorporar cambios regulatorios (NIS2, DORA, etc.) y nuevas buenas prácticas FAIR‑STRIDE.
- Ajustar ponderaciones de dominios y métricas según las prioridades nacionales y sectoriales.
- Compartir lecciones aprendidas en foros profesionales, para converger hacia métricas cada vez más comparables y útiles.

## 6. Nota final

Este kit no pretende ofrecer “la” métrica perfecta, sino un marco razonado para discutir qué medimos, por qué lo medimos y qué hacemos con los resultados. El verdadero éxito no está en la cantidad de indicadores, sino en la calidad de las conversaciones que permiten sostener.
