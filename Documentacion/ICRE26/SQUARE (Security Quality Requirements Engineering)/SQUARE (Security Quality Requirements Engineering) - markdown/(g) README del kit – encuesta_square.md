# README del Kit de Encuesta SQUARE  
## Versión 1.0 – Uso y contenidos

Bienvenido al kit SQUARE de encuesta sobre requisitos de seguridad.  
Si está leyendo esto, probablemente ha aceptado que “seguro” no se define solo en los PowerPoint.

---

## 1. ¿Qué incluye este kit?

1. **Modelo de encuesta integral** (`encuesta_square.md`)  
   El cuestionario completo para aplicar en organizaciones, con preguntas cerradas y abiertas, organizado por bloques temáticos.

2. **Guía metodológica** (`guia_metodologica_square.md`)  
   Instrucciones para aplicar la encuesta, codificar respuestas y calcular el Índice Global de Madurez (IGM) y estimaciones de ROI.

3. **Narrativa explicativa** (`narrativa_square.md`)  
   Contexto, argumentos y relato para explicar a terceros por qué esta encuesta existe y para qué sirve.

4. **Mapeo normativo** (`mapeo_preguntas_normativa_square.md`)  
   Tabla que relaciona cada bloque de preguntas con referencias normativas y marcos de referencia (NIS2, ENS, ISO, ENISA, etc.).

5. **Plantilla de Excel (especificación)** (`plantilla_excel_igm_roi_square.md`)  
   Descripción detallada para que se construya un libro Excel capaz de calcular IGM y ROI a partir de las respuestas.

6. **Plantilla de reporte ejecutivo** (`plantilla_reporte_ejecutivo_square.md`)  
   Esquema de presentación (formato PowerPoint) para llevar los resultados a comités de dirección, consejos o foros sectoriales.

---

## 2. Cómo empezar

1. **Adapte el modelo de encuesta** a su contexto (sector, tamaño, normativa específica).  
2. **Implemente la plantilla Excel** siguiendo la especificación, o construya un panel equivalente en otra herramienta.  
3. **Realice una prueba piloto** con un pequeño grupo de organizaciones voluntarias o unidades internas.  
4. **Ajuste las preguntas y codificación** según las lecciones del piloto.  
5. **Lance la encuesta formal** con un calendario claro y un compromiso de devolución de resultados agregados.

---

## 3. Niveles de uso

Puede usar este kit de varias maneras:

- **Internamente en una sola organización**, para diagnosticar la madurez de sus sistemas críticos.  
- **Sectorialmente**, coordinado por un regulador o asociación empresarial.  
- **A nivel nacional**, como parte de una estrategia de ciberseguridad o de evaluación periódica.

En todos los casos, la recomendación es preservar el anonimato en análisis agregados y compartir resultados comparativos de manera responsable.

---

## 4. Buenas prácticas de implementación

- No convierta la encuesta en un examen de culpabilidad: preséntela como herramienta de mejora.  
- Reserve tiempo para explicar resultados, no solo para recolectar respuestas.  
- Involucre a múltiples perfiles (no solo seguridad) en la cumplimentación.  
- Combine los resultados con datos objetivos (incidentes, auditorías, métricas operativas).  
- Repita el ejercicio de forma periódica para medir progreso.

---

## 5. Licencia de uso intelectual (no apta para abogados estrictos)

Puede adaptar, ampliar y criticar este kit a discreción, siempre que mantenga el espíritu:  
- Promover una ingeniería de requisitos de seguridad más madura y medible.  
- Favorecer la transparencia y la mejora continua.  
- Evitar la tentación de usar los resultados solo para señalar con el dedo.

---

## 6. Contacto y contribuciones

Si descubre mejoras, nuevos indicadores o formas más elegantes de capturar la realidad, compártalas con su comunidad profesional.  
La ingeniería de requisitos de seguridad es un deporte de equipo; este kit no pretende ser dogma, sino punto de partida.
