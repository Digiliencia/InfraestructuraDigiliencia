# README – Kit GQM + PRAGMATIC para indicadores STRIDE

Este kit complementa el “Kit de Encuesta STRIDE” añadiendo una capa explícita de método (GQM) y de evaluación de calidad de métricas (PRAGMATIC).

## 1. Componentes del kit

- `marco_integrativo_gqm_pragmatic.md`  
  Describe objetivos, preguntas y métricas por categoría STRIDE, y cómo se integran GQM y PRAGMATIC.

- `matriz_evaluacion_pragmatic_completa.md`  
  Presenta la puntuación PRAGMATIC de referencia para cada métrica STRIDE.

- `narrativa_explicativa_gqm_pragmatic.md`  
  Desarrolla la lógica y el relato detrás de GQM+PRAGMATIC aplicado a STRIDE.

- `mapeo_gqm_pragmatic_normativo.md`  
  Enlaza métricas con objetivos, preguntas y requisitos normativos.

- `plantilla_excel_igm_roi_gqm_pragmatic.md`  
  Especifica la estructura del libro Excel para implementar cálculos de IGM-STRIDE y ROI basados en estas métricas.

- `plantilla_ppt_reporte_ejecutivo_gqm_pragmatic.md`  
  Propone una estructura de presentación ejecutiva alineada con este marco.

## 2. Flujo de uso recomendado

1. **Definir o validar objetivos** (nacionales, sectoriales, organizativos) en términos STRIDE.
2. **Seleccionar métricas** de entre las propuestas, ajustando sus definiciones técnicas al contexto local.
3. **Evaluar PRAGMATIC** para cada métrica, adaptando la matriz de referencia.
4. **Implantar la captura de datos** en sistemas fuente (SIEM, IAM, CMDB, OT, etc.).
5. **Cargar datos en la plantilla Excel** y calcular índices STRIDE y IGM.
6. **Elaborar el reporte ejecutivo** usando la plantilla PowerPoint.
7. **Revisar periódicamente** objetivos, preguntas y métricas para mantener la relevancia.

## 3. Sinergia con la Encuesta STRIDE

La encuesta proporciona la fotografía declarativa de la madurez; este kit aporta el esqueleto para transformar esa foto en serie temporal medible. Juntos permiten:

- Contrastar percepciones con datos duros.
- Justificar inversiones y priorizaciones frente a dirección y reguladores.
- Alinear el discurso técnico con los objetivos nacionales de ciberseguridad.

## 4. Adaptación y gobierno

Se recomienda designar un propietario organizativo del marco (normalmente, el área de riesgos o el CISO) y establecer un ciclo anual de revisión de:

- Objetivos GQM.
- Métricas activas.
- Evaluaciones PRAGMATIC.

Solo así se evita caer en la tentación de acumular métricas como cromos, sin preguntarse si todavía responden a las preguntas importantes.
