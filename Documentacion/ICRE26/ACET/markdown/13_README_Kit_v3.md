#### 09_Matriz_PRAGMATIC_Completa.md ⭐ EVALUACIÓN INTEGRAL

**Propósito:** Validar calidad de cada indicador contra 9 criterios

**Contenido:**
- 25 indicadores ACET mapeados
- Cada indicador: 9 criterios × 0-3 puntos = Score/27 = %PRAGMATIC
- Interpretación: >90% Excel | 80-90% Bueno | 70-80% Aceptable | <70% Pobre
- Promedio: 90% PRAGMATIC = Cartera robusta

**Ejemplo:**
```
Indicador: MTTD <2 horas
P (Predictivo):  3/3 ✓ (predice daño controlado)
R (Relevante):   3/3 ✓ (CEO vigila)
A (Accionable):  3/3 ✓ (Si >2h = tune SIEM)
G (Genuino):     2/3 ⚠️ (Requiere validar falsos positivos)
S (Significativo):3/3 ✓ (€5K-10K/hora delay)
Pr (Preciso):    3/3 ✓ (Timestamp automático)
O (Oportuno):    3/3 ✓ (Real-time SIEM)
I (Independiente):2/3 ⚠️ (Requiere auditoría externa)
Re (Rentable):   3/3 ✓ (€0 colección)
────────────────────────────
TOTAL: 25/27 = 93% PRAGMATIC ✓ IMPLEMENTAR
```

**Cuándo usar:**
- Priorizar indicadores: Implementar primero >90%
- Validación cruzada: Para <85%, añadir auditoría técnica
- Aprobación inversión: "25 de 25 indicadores validan 90% criterios PRAGMATIC"

#### 11_Plantilla_Excel_Avanzada.md ⭐ IMPLEMENTACIÓN TÉCNICA

**Propósito:** Sistema cálculo integral con 8 hojas especializadas

**Estructura 8 Hojas:**

1. **INPUT**: 60 respuestas + puntos + GQM_level + PRAGMATIC_score
2. **SCORING**: IGM global, índices complementarios (VAS, SES, SATS), KRIs
3. **GQM_MAPPING**: Trazabilidad BG→SG→MG→Q→Metric (para gobernanza)
4. **PRAGMATIC_EVAL**: Matriz 9 criterios × 25 indicadores
5. **BENCHMARKING**: Comparativa sector/nacional + Radar chart
6. **ROI_ANALYSIS**: Controles, inversión, beneficio, payback, ROI%, prioridad
7. **NORMATIVE_MAPPING**: Pregunta → NIST/ENS/NIS2/GDPR/ISO
8. **DASHBOARD_EXECUTIVE**: Visualización resumen (Gauge, Radar, KRIs, Roadmap)

**Fórmulas Clave:**
```
IGM = (GV%×0.15 + ID%×0.15 + PR%×0.25 + DE%×0.20 + RS%×0.15 + RC%×0.10)

PRAGMATIC_Score = SUM(P, R, A, G, S, Pr, O, I, Re) / 27 × 100

ROI% = ((Beneficio - Inversión) / Inversión) × 100

Payback = (Inversión / Beneficio Anual) × 12 meses
```

**Cuándo usar:**
- Importar respuestas encuesta → Excel automático calcula todo
- Reportar IGM + PRAGMATIC scores + ROI en tiempo real
- Compartir dashboard con ejecutivos

**Output:** Fichero Excel profesional listo para distribución

## 3. GOBERNANZA GQM SOSTENIBLE

### Estructura Niveles

```
NIVEL 0: BUSINESS GOAL (Junta/CEO)
├─ Ejemplos: "Mejorar NPS 25%", "Cumplir NIS2", "Reducir riesgos 30%"
├─ Frecuencia Revisión: Anual
├─ Propietario: CEO/CFO
└─ Métrica: Correlación con objetivos ejecutivos

  NIVEL 1: SECURITY GOAL (CISO/CIO)
  ├─ Ejemplos: "MTTD <2h", "Capacitación >95%", "Backups 3-2-1"
  ├─ Frecuencia Revisión: Trimestral
  ├─ Propietario: CISO + IT Director
  └─ Métrica: Alineación con Security Goals

    NIVEL 2: MEASUREMENT GOAL (SOC Lead)
    ├─ Ejemplos: "Cobertura SIEM >85%", "Correlación >60%", "IGM ≥75%"
    ├─ Frecuencia Revisión: Mensual
    ├─ Propietario: SOC Lead + Security Team
    └─ Métrica: Avance vs. Target Measurement Goals

      NIVEL 3: METRICS (Analytics)
      ├─ Ejemplos: MTTD, SIEM_Cov%, Phishing_ClickRate
      ├─ Frecuencia Revisión: Real-time (dashboards) + Semanal (reportes)
      ├─ Propietario: Data Analyst + SOC Analyst
      └─ Métrica: Cumplimiento SLA métrica, PRAGMATIC score >90%
```

### Calendario Operativo

**Mensual:**
- Reunión SOC (Lunes 1er semana): Revisar Metrics, KRIs, alertas
- Output: KPI report para CISO

**Trimestral:**
- Reunión CISO + IT (1a semana): Revisar Measurement Goals, proyección IGM
- Output: Roadmap ajustes, presupuesto remanente

**Anual:**
- Reunión Junta (diciembre): Revisión Business Goals, IGM anual, aprobación presupuesto 2027
- Output: Plan estratégico ciberseguridad 2027

## 4. MEJORES PRÁCTICAS GQM+PRAGMATIC

### Para Recolección Datos

✓ **Hacer:**
- Triangular respuestas: IT + CISO + Negocio (3 perspectivas mínimo)
- Validar PRAGMATIC score con auditor técnico (para <85%)
- Documentar supuestos y limitaciones
- Permitir "DK" (Desconoce) sin penalizar

✗ **NO hacer:**
- Aceptar "Sí" sin evidencia
- Mezclar niveles GQM (pregunta debe mapear a single level)
- Cambiar preguntas mid-stream
- Penalizar respondentes por DK alto

---

### Para Análisis

✓ **Hacer:**
- Validar IGM con scan técnico independiente (Nessus, etc.)
- Para PRAGMATIC <85%, incluir auditoría cruzada
- Reconciliar discrepancias entre respondentes
- Investigar "outliers" (ej. "Sí SIEM" pero "No" correlación)

✗ **NO hacer:**
- Asumir linealidad: IGM 71% ≠ Riesgo 29% (NO son opuestos)
- Comparar directamente con competidores (contextos distintos)
- Publicar scores individuales por respondente (privacidad)
- Ignorar "NA" (puede indicar falta de visibilidad)

---

### Para Gobernanza

✓ **Hacer:**
- Asignar propietarios explícitos para cada nivel GQM
- Calendaricar revisiones: Real-time (Metrics) → Anual (Goals)
- Conectar compensation/incentives a SGs/MGs (si aplica)
- Documentar decisiones, cambios, excepciones

✗ **NO hacer:**
- Dejar métricas sin dueño (→ abandono)
- Revisar solo IGM global (perder granularidad)
- Cambiar ponderaciones de dominios sin documentar razón
- Implementar 50 iniciativas sin priorizar (parálisis)

## 5. SOLUCIÓN DE PROBLEMAS

### Problema: PRAGMATIC Score Bajo (<75%) para Métrica Importante

**Diagnóstico:**
```
Causas Comunes:
1. Métrica "Genuina" baja: ¿Mide qué pretende o hay sesgo?
2. Métrica "Independiente" baja: ¿Verificable objetivamente?
3. Métrica "Oportuno" baja: ¿Datos disponibles rápido?
```

**Soluciones:**
1. Añadir validación técnica independiente (ej. auditor externo)
2. Automatizar colección (reducir sesgo manual)
3. Documentar "Notas de Auditoría" (limitaciones conocidas)
4. Aceptar si score >70% + hay compensación otras métricas

---

### Problema: GQM Levels No Alineados (Pregunta mapea a 2 niveles)

**Diagnóstico:**
```
Ejemplo: P4.2.1 ("¿MTTD <2h?")
¿Mapea a?: MG (Measurement Goal) o Q (Question)?
→ Ambigüedad
```

**Solución:**
1. Clarificar: MTTD es MÉTRICA (Nivel 3), no Question
2. Question sería: "¿Detectamos ataques en <2 horas?"
3. Métrica: "MTTD = [timestamp diff] en horas"
4. Documentar distinción en 08_Marco_GQM_PRAGMATIC.md

---

### Problema: ROI Analysis Muestra Payback >12 Meses

**Diagnóstico:**
```
Ejemplo: SIEM costó €150K, beneficio €180K → Payback 10 meses ✓
Pero EDR costó €120K, beneficio €80K → Payback 18 meses ❌
```

**Decisión:**
1. Si payback >12 meses: Cuestionar justificación
2. Opciones:
   a. Rechazar iniciativa (no justificable ROI económico)
   b. Aceptar por razón regulatoria (ej. GDPR compliance)
   c. Restructurar: Fase 1 (€50K, payback 8m) + Fase 2 (€70K, posterior)

## 6. FAQ

### ¿Puedo usar solo archivos 01-07 (Kit ACET original)?

**Respuesta:** Sí, funciona. Pero pierde:
- Trazabilidad Goal→Metric (no sabes por qué cada métrica)
- Validación PRAGMATIC (no sabes si métrica es realmente útil)
- Alineación negocio (dificultad justificar presupuesto)

Recomendación: Mínimo leer 08_Marco + usar 09_Matriz para validación.

---

### ¿Qué diferencia hay entre GQM original (Basili) y GQM+?

**Respuesta:**

| Aspecto | GQM Original | GQM+ Strategies |
|---------|-------------|-----------------|
| Niveles | 3 (Goal, Q, M) | 4 (BG, SG, MG, Q/M) |
| Alineación | Local (equipo) | Global (empresa) |
| Uso | Medición táctico | Gobernanza estratégica |
| Implementación | Medio | Alto |

Este kit usa **GQM+**, más ambicioso pero más justificable.

---

### ¿Cuánto tiempo requiere implementar todo?

**Respuesta:**

| Fase | Tiempo | Personas |
|-----|--------|----------|
| Investigación (Leer marcos) | 5h | 2 (CISO, Analyst) |
| Preparación (Configurar Excel) | 10h | 1 (Analyst) |
| Recolección (Encuesta) | 15h | 3 (Distribución + seguimiento) |
| Análisis (Cálculos + validación) | 20h | 2 (Analyst + Auditor) |
| Reportes (PowerPoint + narrativa) | 15h | 1 (Analyst) |

**Aceleración:**
- Solo 01_Encuesta + 02_Guia + 05_Excel: 30h (rápido pero menos profundo)

### ¿Es obligatorio usar Excel avanzada (11_Plantilla)?

**Respuesta:** No, alternativas:

1. **Google Sheets** (similar estructura, gratis, colaborativo)
2. **Power BI** (si ya tenéis, más visual)
3. **Salesforce** (si CRM store datos)
4. **Manual Excel** (usar 05_Plantilla básica + cálculos manuales)

Recomendación: 11_Plantilla avanzada si >3 usuarios, >50 preguntas, necesita automatización.
