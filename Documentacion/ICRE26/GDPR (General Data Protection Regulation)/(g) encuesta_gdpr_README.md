# README
## Kit de Encuesta GDPR – Ciberseguridad, Riesgo y Métricas

Bienvenido al pequeño laboratorio donde el RGPD, la ciberseguridad y la estadística
deciden hablarse sin intermediarios.

Este kit contiene:

1. `encuesta_gdpr_modelo.md`  
   Modelo de encuesta integral para responsables de organizaciones.

2. `encuesta_gdpr_guia_metodologica.md`  
   Guía para diseñar la aplicación, puntuación e interpretación de resultados.

3. `encuesta_gdpr_narrativa.md`  
   Narrativa explicativa para contextualizar y “vender” internamente el ejercicio.

4. `encuesta_gdpr_mapeo_normativo.md`  
   Tabla que mapea cada bloque/pregunta a artículos RGPD y guías oficiales.

5. `encuesta_gdpr_igm_roi.md`  
   Plantilla conceptual para construir un Excel de cálculo de IGM y ROI.

6. `encuesta_gdpr_plantilla_ppt.md`  
   Guion de presentación ejecutiva para consejo/alta dirección.

---

## Cómo usar el kit (en tres actos)

### Acto I – Diagnóstico

1. Adaptar el modelo de encuesta al contexto de la organización (sector, tamaño).
2. Decidir formato de aplicación (online, entrevistas, Excel).
3. Informar de forma transparente sobre el tratamiento de las respuestas. [web:29][web:18]
4. Ejecutar la encuesta con los perfiles adecuados (CISO, DPO, Riesgos, TI, etc.).

### Acto II – Análisis

1. Volcar las respuestas en el Excel diseñado a partir de `encuesta_gdpr_igm_roi.md`.
2. Calcular:
   - IGM por bloque.
   - IGM global.
   - Si se desea, ROI aproximado de iniciativas clave.
3. Revisar el mapeo normativo para comprender qué está fallando y dónde. [web:28][web:25][web:14]

### Acto III – Acción

1. Construir el reporte ejecutivo siguiendo `encuesta_gdpr_plantilla_ppt.md`.
2. Presentar resultados a la alta dirección, con énfasis en:
   - Riesgos.
   - Oportunidades de mejora.
   - Relación coste/beneficio de las iniciativas propuestas. [web:12][web:22]
3. Integrar indicadores seleccionados en cuadros de mando regulares.

## Requisitos mínimos de cordura

Para que el kit funcione razonablemente bien, se recomienda:

- Aceptar que algunas respuestas serán incómodas.
- Evitar “maquillar” la realidad para quedar bien en el informe.
- Revisar periódicamente (al menos anual) la encuesta, actualizando contenidos según:
  - evolución normativa (NIS2, guías EDPB, AEPD, ENISA), [web:22][web:26][web:14][web:21]
  - aparición de nuevas tecnologías (IA generativa, espacios de datos, etc.).

## Crédito intelectual y fuentes

El enfoque se inspira en:

- El RGPD y sus artículos sobre seguridad del tratamiento, brechas y responsabilidad. [web:28]
- El digest de casos del EDPB sobre seguridad y brechas. [web:14]
- Las guías de ENISA y AEPD sobre seguridad de tratamiento y criptografía. [web:25][web:21]
- Los modelos de madurez NIS2 (NIS360). [web:22][web:26]
- El mapeo práctico entre RGPD, ISO 27001 e ISO 27701. [web:23][web:27]
- Informes sobre brechas, sanciones y costes globales. [web:12][web:3][web:6][web:11]

## Licencia de uso (informal, pero seria)

Puede adaptar libremente este kit para:

- uso interno en su organización,
- proyectos de consultoría,
- investigaciones y estudios comparativos.

Se agradece citar la procedencia conceptual (sin necesidad de solemnidades) y,
sobre todo, mantener el espíritu: rigor normativo, aterrizaje técnico y un
mínimo de ironía para navegar la complejidad sin perder la salud mental.

## Última recomendación

Antes de enviar la encuesta, hágase esta pregunta:

> “¿Respondería yo mismo con honestidad brutal a esta encuesta si fuera el CISO?”

Si la respuesta es sí, adelante. Si la respuesta es no, todavía está a tiempo
de ajustar tono, alcance o destinatarios.
