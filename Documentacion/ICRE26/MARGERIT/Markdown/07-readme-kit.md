## ¿QUÉ ES ESTE KIT?

Este es un **conjunto profesional y descargable de documentos** (7 archivos .md) que operacionaliza:
- **MAGERIT v3** (metodología española análisis riesgos)
- **Directiva NIS2** (regulación ciberseguridad UE)
- **ENS RD 311/2022** (esquema nacional seguridad españa)
- **NIST CSF 2.0** (marco internacional flexible)
- **ISO 27001:2022** (estándar SGSI certificable)

Todo en **una encuesta integral de 75+ preguntas** que evalúa madurez ciberseguridad organizacional y genera:
✅ Índice Global de Madurez (IGM %) con nivel CMM L0-L5  
✅ Identificación brechas regulatorias (NIS2, ENS)  
✅ Cuantificación riesgos residuales (€ ALE/año)  
✅ Análisis ROI de inversiones seguridad  
✅ Recomendaciones priorizadas con presupuesto

### **¿Es obligatoria la encuesta para cumplir NIS2?**

No obligatoria formalmente, pero **altamente recomendada:**
- NIS2 requiere "análisis riesgos formal" (Art 21a) = MAGERIT MAR
- Encuesta operacionaliza MAGERIT de forma estructurada
- Auditor externo solicita evidencia análisis riesgos = documentos encuesta + informe

**Conclusión:** No es legal requirement, pero es best practice para auditoría.

### **¿Puedo adaptar las preguntas para mi sector (ej. Sanidad, Finanzas)?**

Sí. Recomendaciones:
1. Mantener estructura base (10 bloques, 75 preguntas)
2. Adaptar en secciones:
   - Bloque 7 (Resiliencia): **Sanidad** añadir RTO <15min (crítico emergencias)
   - Bloque 3 (Métricas): **Finanzas** especificar CVSS + cumplimiento regulatorio
   - Bloque 4 (Vulnerabilidades): **Energía/ICS** añadir pruebas OT/SCADA
3. Crear version "sector-specific" documento palabra clave

**Ej.:** `01-encuesta-integral-SANIDAD-v2.md` con ajustes personalizados

---

### **¿Con qué frecuencia debo repetir la encuesta?**

Recomendación:
- **Completa (75 preguntas):** Anual (Q1 o Q4)
- **Mini-assessment (20 preguntas):** Trimestral (bloques críticos: MAR, SIEM, Incidentes)
- **Ad-hoc:** Post-incidente o cambio legislativo significativo

**Rationale:** Detectar tendencias, validar plan acción, adaptar a nuevas amenazas.

---

### **¿Qué hago si respondentes dan respuestas muy diferentes (ej. CISO L4 SIEM, TI L1)?**

1. **Análisis:** Discrepancia es dato válido. Indica desalineación interno.
2. **Sesión clarificación:** CISO + TI revisar preguntas específicas (5.1-5.4)
3. **Consenso:** Revisar evidencia (demo SIEM, logs, alertas reales)
4. **Ajuste:** Usar media respuestas para análisis, pero registrar observación cualitativa
5. **Acción:** "Alinear percepción seguridad SIEM entre liderazgo" → plan mejora

**Conclusión:** No es problema, es oportunidad identificar gaps comunicación.

---

### **¿Cómo calculo ALE/ROI si no tengo histórico incidentes?**

Opciones:
1. **Benchmarking sectorial:** Usar tasas INCIBE (España: ~1,5 incidentes/año por empresa promedio)
2. **FAIR simulación:** Expertos opinión (TEF estimada, LM comparable sector)
3. **Conservative:** Asumir TEF baja, LM moderada → ALE mínimo
4. **Expert panel:** 3 expertos estiman por votación, media ponderada

**Recomendación:** Combinar benchmarking + expert opinion, documentar supuestos.
