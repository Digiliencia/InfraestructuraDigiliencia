## INDICADORES GQM+PRAGMATIC: EJEMPLO APLICADO

### Indicador: "Cobertura de Salvaguardas por Amenaza Crítica"

**NIVEL 1: GOAL**
```
"REDUCIR riesgo residual no mitigado <€500K/año 
 para DEMOSTRAR conformidad NIS2 a Junta Directiva 
 en 12 meses"
```

**NIVEL 2: QUESTION**
```
"¿Qué % de amenazas críticas tiene salvaguarda asignada?"
```

**NIVEL 3: METRIC**
```
% Cobertura Salvaguardas 
= (# Amenazas con salvaguarda) / (Total amenazas críticas) × 100

Fórmula: VLOOKUP(amenaza, matriz_amenaza_salvaguarda) → 1/0
Fuente: CMDB + Herramienta PILAR MAGERIT
Frecuencia: Trimestral
Responsable: CISO
```

**NIVEL 4: VALIDACIÓN PRAGMATIC**

| Criterio | Puntuación | Justificación |
|----------|-----------|--------------|
| **P**redictivo | 5/5 | Gap salvaguardas predice brecha riesgo futuro (R²=0.82) |
| **R**elevante | 5/5 | NIS2 Art 21 obliga medidas protección |
| **A**ccionable | 5/5 | Si <80%, CISO asigna recursos; acción ejecutable |
| **G**enuino | 5/5 | Auditor valida control existe (sí/no binario) |
| **S**ignificativo | 4/5 | Gap 10% = típicamente 1-2 controles faltantes = €100K+ ALE |
| **P**reciso | 4/5 | Conteo binario; error <3% |
| **O**ortuno | 5/5 | Datos real-time CMDB |
| **I**ndependiente | 5/5 | Ortogonal a otras métricas |
| **R**entable | 5/5 | 5h/año, 0€ inversión |
| **PRAGMATIC Score** | **4.7/5.0** | ✓ EXCELENTE → Referencia benchmarking |

**ACCIÓN RECOMENDADA:** Mantener indicador. Máxima prioridad implementación.

## PREGUNTAS FRECUENTES: GQM+PRAGMATIC

### **¿Cómo difiere esto de KPIs tradicionales?**
- **Tradicional:** "Tenemos 50 KPIs" (sin trazabilidad GOAL→METRIC)
- **GQM+PRAGMATIC:** "10 indicadores críticos PRAGMATIC >4.0, alineados a 5 GOALs estratégicos, auditables"

### **¿Cuál es el overhead administrativo?**
- Diseño inicial: 40h (month 1)
- Recolección datos: 10h/mes
- Análisis+reporte: 5h/mes
- **Total anual:** ~150h (< 1 FTE 20% dedicación)

### **¿Cómo aseguro que PRAGMATIC score sea justo?**
- Auditoría externa anual (€10-20K)
- Múltiples evaluadores (3 mínimo) → consenso
- Triangulación datos (2+ fuentes)
- Documentación completa (audit trail)

### **¿Cuándo reediseño indicadores débiles (<3.5)?**
- Piloto 3 meses
- Si PRAGMATIC score no mejora después ajustes, rediseñar
- Ciclo anual de mejora (revisión Q4)
