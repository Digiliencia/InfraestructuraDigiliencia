# README – Kit de Encuesta VAST Nacional (España)

Bienvenido/a al kit de encuesta VAST. Si ha llegado hasta aquí, es probable que le hayan convencido (o que usted haya convencido a otros) de que la intuición, por sí sola, no basta para gestionar ciber riesgos.

---

## 1. Contenido del kit

El kit incluye los siguientes componentes (formato Markdown, listos para adaptar):

1. `encuesta_vast_integral.md`  
   Modelo de encuesta completo para responsables de organizaciones.
2. `guia_metodologica_vast.md`  
   Explica objetivos, población, diseño, cálculo del IGM y ROI.
3. `narrativa_vast.md`  
   Relato explicativo para acompañar la encuesta y los resultados.
4. `mapeo_preguntas_normativa.md`  
   Tabla que relaciona cada pregunta con requisitos normativos (NIS2, DORA, ENS, ISO, NIST CSF).
5. `plantilla_excel_igm_roi.md`  
   Instrucciones para crear una hoja de cálculo con IGM y ROI.
6. `plantilla_reporte_ejecutivo_ppt.md`  
   Guion para preparar una presentación de resultados a nivel ejecutivo.

---

## 2. Uso recomendado

1. **Adaptar la encuesta** (`encuesta_vast_integral.md`) al contexto sectorial y de idioma interno (si procede).
2. **Validar la guía metodológica** (`guia_metodologica_vast.md`) con el equipo de riesgo, cumplimiento y/o reguladores implicados.
3. **Configurar el Excel** siguiendo `plantilla_excel_igm_roi.md` y probarlo con un conjunto de respuestas de prueba.
4. **Pilotar la encuesta** con algunas organizaciones para verificar comprensión, tiempos y relevancia de preguntas.
5. **Definir un calendario** de despliegue de la encuesta y de reportes de resultados (anual, bienal, etc.).
6. **Preparar el relato** (`narrativa_vast.md`) para acompañar tanto la invitación a participar como la presentación de conclusiones.

---

## 3. Personalización

El kit está pensado como base. Se recomienda:

- Ajustar el lenguaje a la cultura de su organización o sector (manteniendo, si se puede, el tono humano).
- Añadir o quitar preguntas según el marco normativo aplicable.
- Ajustar la ponderación de bloques del IGM (bloques THR, MET, RES pueden tener mayor peso en contextos muy operativos).
- Afinar el cálculo de ROI con sus propios datos contables y de pérdidas históricas.

---

## 4. Consideraciones sobre confidencialidad y ética

- Defina claramente quién verá los resultados individuales y quién verá solo agregados.
- Evite utilizar la encuesta como herramienta de sanción interna; úsela como base de diálogo y mejora.
- Explique con transparencia a las organizaciones participantes qué uso se dará a sus datos y cómo se anonimizarán.

---

## 5. Próximos pasos sugeridos

Una vez realizado el primer ciclo de encuesta:

1. Analizar los resultados agregados y por sector.
2. Compartir con las organizaciones participantes un informe comparativo (benchmark) con recomendaciones.
3. Usar las conclusiones para priorizar inversiones a nivel organizativo y, si aplica, nacional.
4. Planificar el siguiente ciclo (por ejemplo, al año siguiente) para observar tendencias.

---

## 6. Contacto y mantenimiento

Idealmente, este kit debería tener:

- Un “propietario” institucional (por ejemplo, un CERT nacional, una asociación sectorial o un regulador).
- Un ciclo de revisión anual para:
  - Actualizar referencias normativas.
  - Ajustar preguntas a nuevos riesgos (p. ej. IA generativa, nuevas regulaciones).
  - Incorporar lecciones aprendidas del uso real.

Mientras tanto, si ha llegado leyendo hasta aquí, felicidades: está oficialmente por encima de la media en paciencia metodológica.
