# README – Kit GQM + PRAGMATIC para Indicadores VAST Nacionales

## 1. Contenido del kit

Este kit complementa la encuesta VAST nacional con un marco estructurado de objetivos, métricas y evaluación de calidad:

1. `marco_gqm_pragmatic.md` – Definición del marco integrativo GQM + PRAGMATIC.
2. `matriz_pragmatic_completa.md` – Tabla de evaluación PRAGMATIC de las principales métricas.
3. `narrativa_gqm_pragmatic.md` – Relato explicativo para contextualizar el uso del marco.
4. `mapeo_preguntas_normativa_gqm.md` – Mapeo de objetivos/métricas a marcos normativos (NIS2, DORA, ENS, ISO, NIST CSF).
5. `plantilla_excel_gqm_pragmatic.md` – Estructura sugerida de Excel para gestionar métricas GQM + PRAGMATIC, IGM y ROI.
6. `plantilla_reporte_ejecutivo_gqm.md` – Guion de presentación ejecutiva para comunicar el marco y resultados.

## 2. Cómo utilizar este kit

1. **Acordar objetivos (G)** a nivel nacional o sectorial, usando el documento de marco como base.
2. **Validar el catálogo de métricas (M)** y la matriz PRAGMATIC con un grupo de expertos (CISOs, reguladores, analistas de riesgo).
3. **Implementar la plantilla Excel** para consolidar datos de encuesta, calcular IGM y analizar la calidad de métricas.
4. **Utilizar el mapeo normativo** para justificar la pertinencia de las métricas ante auditorías y supervisores.
5. **Apoyarse en el reporte ejecutivo** para explicar a la alta dirección por qué se han elegido esas métricas y qué decisiones habilitan.

## 3. Personalización

- Los objetivos G1–G6 pueden ampliarse o refinarse para sectores concretos (sanidad, energía, finanzas, sector público…).
- Las puntuaciones PRAGMATIC son orientativas; es recomendable revisarlas periódicamente según experiencia.
- Se pueden introducir pesos adicionales para criterios PRAGMATIC según las prioridades (por ejemplo, enfatizar la “Accionabilidad” y la “Rentabilidad”).

## 4. Buenas prácticas

- Mantener el catálogo de métricas manejable; priorizar calidad sobre cantidad.
- Documentar cualquier cambio en definiciones de métricas para preservar comparabilidad temporal.
- Comunicar de forma transparente el uso que se hará de los datos: diagnóstico y mejora, no caza de brujas.

## 5. Próximos pasos sugeridos

1. Lanzar un piloto con un pequeño conjunto de organizaciones voluntarias.
2. Recoger feedback sobre dificultades de medición y carga de trabajo.
3. Ajustar métricas, periodicidad y puntuaciones PRAGMATIC.
4. A partir de ahí, planificar el despliegue a mayor escala.

Este kit pretende, en suma, que cada número que aparezca en un informe de ciberseguridad tenga una razón de ser, una pregunta a la que responde y una decisión que ayuda a tomar.
