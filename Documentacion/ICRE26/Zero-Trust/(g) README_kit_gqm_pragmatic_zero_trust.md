# README – Kit GQM + PRAGMATIC para indicadores Zero Trust

## 1. Contenido del kit

Este kit complementa el "Kit de Encuesta de Madurez Zero Trust" añadiendo un marco formal para alinear objetivos, preguntas y métricas (GQM) y para evaluar la calidad de las métricas (PRAGMATIC).

Incluye:

- `marco_gqm_pragmatic_zero_trust.md`: descripción del marco integrativo GQM + PRAGMATIC aplicado a Zero Trust.  
- `matriz_pragmatic_zero_trust.md`: evaluación PRAGMATIC de las métricas clave.  
- `narrativa_gqm_pragmatic_zero_trust.md`: texto explicativo para contextualizar el uso del marco.  
- `mapeo_preguntas_normativa_gqm_zero_trust.md`: tabla de mapeo detallado entre preguntas, objetivos GQM y normativa.  
- `plantilla_excel_gqm_pragmatic_zero_trust.md`: especificación de un libro Excel que integra IGM, métricas GQM y PRAGMATIC.  
- `plantilla_reporte_ejecutivo_gqm_pragmatic_zero_trust.md`: guía para preparar presentaciones ejecutivas.  

## 2. Requisitos de uso

- Haber desplegado (o estar desplegando) la encuesta de madurez Zero Trust.  
- Disponer de un mínimo de capacidades de análisis de datos (Excel u otra herramienta).  
- Contar con apoyo de las áreas de seguridad, riesgo y, preferiblemente, de reguladores o asociaciones sectoriales para la interpretación conjunta.

## 3. Flujo de trabajo recomendado

1. **Definición de objetivos**: revisar los objetivos GQM propuestos y adaptarlos a la realidad del país, sector u organización.  
2. **Selección de métricas**: usar la matriz PRAGMATIC para priorizar las métricas que entrarán en el cuadro de mando principal.  
3. **Implementación en Excel**: construir o adaptar el libro Excel siguiendo la plantilla proporcionada.  
4. **Recogida de datos**: volcar resultados de la encuesta y, cuando sea posible, datos técnicos adicionales de sistemas (IAM, SIEM, etc.).  
5. **Análisis y reporting**: calcular IGM, métricas GQM y, si procede, ROI; generar reportes ejecutivos con la plantilla de presentación.  
6. **Revisión periódica**: actualizar objetivos y métricas GQM, así como puntuaciones PRAGMATIC, cada 12–24 meses.

## 4. Relación con ENS, NIS2, DORA y otros marcos

El marco GQM + PRAGMATIC está diseñado para:

- Facilitar la justificación de métricas ante auditores y supervisores ("esta métrica responde a este objetivo normativo"). [web:53][web:47]  
- Alinear indicadores técnicos con requisitos de gestión de riesgos, continuidad y gobernanza de seguridad.  
- Servir de base para cuadros de mando nacionales (observatorios de confianza digital, informes país, etc.). [web:41]

## 5. Licencia y adaptabilidad

Este kit puede adaptarse a distintos contextos nacionales y sectoriales, manteniendo las ideas centrales de GQM y PRAGMATIC. Se recomienda citar las fuentes conceptuales (NIST, ENISA, literatura de métricas de seguridad) y la procedencia del marco cuando se utilice en informes públicos o académicos. [web:51][web:53]
