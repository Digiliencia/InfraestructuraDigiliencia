## 🚀 GUÍA DE USO RÁPIDO (4 SEMANAS)

### Semana 1: PREPARACIÓN Y LANZAMIENTO

```
Tarea 1: Leer el marco (A1-Marco-GQM-PRAGMATIC.md)
  └─ Tiempo: 2 horas
  └─ Propósito: Entender GQM + PRAGMATIC

Tarea 2: Seleccionar respondientes
  └─ Perfil: CISO, CTO, Managers, Técnicos
  └─ Cantidad: 6-9 personas
  └─ Criterio: Representantes de 3+ departamentos

Tarea 3: Briefing a equipo
  └─ Mensaje: "Mejora, no culpa"
  └─ Propósito: Diagnóstico honesto
  └─ Tiempo: 1 hora sesión
  
Tarea 4: Configurar encuesta
  └─ Usar Survey123, Google Forms, o manual Excel
  └─ Tiempo estimado respuesta: 90-120 minutos
  └─ Fecha límite: Fin semana 1
```

### Semana 2: RECOLECCIÓN Y ANÁLISIS

```
Tarea 1: Consolidar respuestas
  └─ Verificar completitud (112 preguntas × 9 respondientes)
  └─ Validar scores (1-5 únicamente)
  └─ Crear base de datos RAW_DATA

Tarea 2: Implementar plantilla Excel (A4)
  └─ Crear 8 sheets según especificación
  └─ Ingresar datos RAW_DATA
  └─ Verificar fórmulas (sin errores circulares)

Tarea 3: Calcular indicadores
  └─ IGM, ICN, NIST Level, ROICS, Payback
  └─ Generar gráficos (radar, barras, líneas)
  └─ Exportar dashboard

Tarea 4: Validación cruzada
  └─ Revisar percentiles vs benchmarks (A2)
  └─ Confirmar PRAGMATIC scores
  └─ Checkear mapeo normativo (A3)
```

### Semana 3: ANÁLISIS Y REPORTE

```
Tarea 1: Análisis detallado por dominio
  └─ Identificar 3-5 dominios críticos
  └─ Documentar hallazgos principales
  └─ Calcular brechas vs objetivos

Tarea 2: Redactar recomendaciones
  └─ Top 5 acciones prioritarias
  └─ Impacto financiero (ROICS)
  └─ Timeline realista

Tarea 3: Mapear conformidad regulatoria (A3)
  └─ % NIS2 actual vs objetivo
  └─ Huecos ISO 27001
  └─ Brecha NIST CSF

Tarea 4: Documento interim
  └─ Borrador informe ejecutivo
  └─ Tablas y gráficos iniciales
  └─ Revisar con CISO
```

## 💡 CÓMO INTERPRETAR RESULTADOS

### Índice Global de Madurez (IGM: 0-5)

```
IGM < 1.5    → TIER 1 EMERGENTE
             Riesgo: Crítico
             Acción: Transformación completa
             Timeline: 24 meses

IGM 1.5-2.5  → TIER 2 BÁSICO
             Riesgo: Alto
             Acción: Plan de mejora urgente
             Timeline: 18-24 meses

IGM 2.5-3.5  → TIER 3 ESTABLECIDO ← TÍPICAMENTE AQUÍ
             Riesgo: Moderado-alto
             Acción: Optimización necesaria
             Timeline: 12-18 meses

IGM 3.5-4.5  → TIER 4 GESTIONADO (OBJETIVO)
             Riesgo: Moderado
             Acción: Mejora continua
             Timeline: 12+ meses

IGM > 4.5    → TIER 5 OPTIMIZADO (LIDERAZGO)
             Riesgo: Bajo
             Acción: Investigación/innovación
             Timeline: N/A
```

### Conformidad Normativa (ICN: %)

```
ICN ≥ 85%    → CONFORME (Auditoría probable aprobación)
ICN 70-85%   → PARCIALMENTE CONFORME (Gap mínimo)
ICN < 70%    → NO CONFORME (Riesgo regulatorio)

ACCIÓN POR SCORE:
- <60%: CRISIS → Corrección urgente antes auditoría
- 60-75%: CRÍTICA → Plan de conformidad 90 días
- 75-85%: IMPORTANTE → Mejorar antes año fiscal
- ≥85%: CONFORME → Mantener y auditar
```

### ROICS y ROI

```
ROICS > 150% → EXCEPCIONAL (Caso de éxito)
ROICS 100-150% → EXCELENTE (Altamente recomendado)
ROICS 50-100% → MUY BUENO (Fiscalmente justificable)
ROICS 25-50% → BUENO (Beneficio combinado)
ROICS 0-25% → REVISAR (Requiere redefinición)
ROICS < 0% → NO RECOMENDABLE (Recalcular)

CONTEXTO NIS2:
Incluso si ROICS fuera negativo, conformidad sería
OBLIGATORIA (no es decision fiscal). ROI simplemente
lo hace más justificable a Board.
```

### PRAGMATIC Score de Métricas

```
PRAGMATIC ≥ 4.5 → EXCEPCIONAL (Implementar sin dudas)
PRAGMATIC 4.0-4.5 → EXCELENTE (Implementar prioritario)
PRAGMATIC 3.5-4.0 → BUENO (Implementar con notas)
PRAGMATIC 3.0-3.5 → ACEPTABLE (Considerar mejoras)
PRAGMATIC < 3.0 → DÉBIL (Redefinir o descartar)

EN ESTE KIT: 
✓ 5 métricas EXCEPCIONALES (≥4.8)
✓ 11 métricas EXCELENTES (4.0-4.8)
⚠ 2 métricas ACEPTABLES (3.5-4.0)
→ 100% recomendadas para implementar
```

## ❓ PREGUNTAS FRECUENTES (FAQs)

### P: ¿Cuál es la diferencia con otras evaluaciones de ciberseguridad?

**R:** Este kit combina tres ventajas únicas:
1. **GQM trazabilidad**: Cada métrica vinculada a objetivo estratégico (no aisladas)
2. **PRAGMATIC validación**: Cada métrica evaluada contra 9 criterios de utilidad
3. **Financiero explícito**: Cálculos ROICS transparentes (no solo risk score)

Resultado: Evaluación honesta + accionable + justificable financieramente.

---

### P: ¿Cuánto tiempo toma completar todo el proceso?

**R:** 4 semanas intensas:
- Semana 1: Preparación (20 horas)
- Semana 2: Recolección/análisis (30 horas)
- Semana 3: Análisis detallado (25 horas)
- Semana 4: PowerPoint/presentación (20 horas)
**Total: ~95 horas team effort (puede distribuirse 3-5 personas)**

---

### P: ¿Es obligatorio hacer todos los 6 documentos?

**R:** No, pero depende del objetivo:
- **Mínimo (para decisión interna)**: A4 (Excel) + parte de A5 (slides)
- **Recomendado (para auditoría)**: A1-A6 (todo el kit)
- **Máximo (para governance)**: A1-A6 + externos (auditor, consultor)

---

### P: ¿Cuál es el costo típico del kit?

**R:** 
- **Herramientas**: €0 (solo Excel/PowerPoint)
- **Servicios**: €0 (DIY) a €50K (consultor externo)
- **Tiempo interno**: 95 horas × €150/hora (típico CISO) = €14.25K costo de oportunidad

**TOTAL**: €14K DIY a €64K con consultor

**COMPARATIVA**: Auditoría externa completa = €80-150K → Este kit es 10x más económico

---

### P: ¿Qué pasa si mi IGM es muy bajo (<2.0)?

**R:** No es vergüenza, es diagnóstico:
1. Nombrar CISO dedicado (si no existe)
2. Documentar políticas básicas (2-3 meses)
3. Implementar herramientas SIEM/escaneo (4-6 meses)
4. Capacitación básica (continuo)
5. Re-evaluar en 12 meses

**Benchmarks muestran**: TIER 1 → TIER 3 típicamente 18-24 meses.

---

### P: ¿Cómo vendo esto a la Junta si no entienden ciberseguridad?

**R:** Usa estos argumentos:
1. **Regulatorio**: "Multa NIS2 potencial €7M vs. inversión €450K"
2. **Financiero**: "ROI 114% en 3 años (mejor que bonos)"
3. **Competitivo**: "Posición 0.4 puntos ARRIBA del mercado"
4. **Riesgos operacionales**: "Breach típico cuesta €2.5M en España"

→ En PowerPoint, enfatizar números, no tecnología.

---

### P: ¿Cuándo re-evaluar (volver a hacer la encuesta)?

**R:** Recomendado:
- **Métricas críticas (MTTD, MTTR)**: Mensuales
- **Dominios críticos (SIEM, NORM)**: Trimestrales
- **Evaluación completa**: Anual

→ Ciclo de mejora continua (PDCA: Plan-Do-Check-Act)

---

### P: ¿Qué pasa si no tengo presupuesto?

**R:** Opciones (en orden):
1. **Faseado**: Críticas primero (€230K Q1) → resto gradual
2. **MSSP (Managed Security Service Provider)**: Externalizar → reduce costo operacional
3. **Herramientas gratuitas**: OpenScap, Wazuh (SIEM open-source), Nuclei (pentest)
4. **Justificar con ROICS**: "ROI 114% permite autofinanciamiento" → Pedir préstamo/inversión

---

### P: ¿Es confidencial el reporte?

**R:** Sí. Contiene información sensible:
- Vulnerabilidades identificadas
- Brechas de controles
- Decisiones de gestión de riesgos

**Recomendado:**
✓ Almacenar en servidor seguro (encriptado)
✓ Acceso restringido a CISO, CFO, Director
✓ No circular en email sin cifrado
✓ Destruir borradores post-finalización
