# README – Kit GQM + PRAGMATIC para Métricas SASE

Bienvenido al **Kit GQM + PRAGMATIC para indicadores SASE / Zero Trust**. Esta colección de documentos está diseñada para ayudar a alinear objetivos nacionales y corporativos con métricas técnicas SASE, y a filtrar esas métricas con criterios de calidad.

---

## 1. Contenido del kit

- `marco-integrativo-gqm-pragmatic.md`  
  Marco conceptual que enlaza Objetivos → Preguntas → Métricas SASE y explica cómo aplicar PRAGMATIC a cada métrica.

- `matriz-evaluacion-pragmatic-completa.md`  
  Tabla con las métricas clave SASE y una puntuación orientativa en cada dimensión PRAGMATIC.

- `narrativa-explicativa-gqm-pragmatic.md`  
  Texto explicativo para contextualizar el uso del marco ante CISOs, responsables de continuidad, riesgos y dirección.

- `mapeo-preguntas-gqm-normativa.md`  
  Tabla que vincula las preguntas/métricas del modelo con requisitos ENS, NIS2 y otros marcos relevantes.

- `plantilla-excel-igm-roi-gqm-pragmatic.md`  
  Descripción de cómo implementar en Excel el IGM y el ROI SASE usando las métricas GQM/PRAGMATIC.

- `plantilla-reporte-ejecutivo-gqm-pragmatic.md`  
  Esqueleto de presentación ejecutiva para comunicar resultados de IGM, ROI y hallazgos a comités de dirección o reguladores.

---

## 2. Orden recomendado de uso

1. **Leer la narrativa (`narrativa-explicativa…`)** para entender la lógica GQM + PRAGMATIC.  
2. **Revisar el marco (`marco-integrativo…`)** y, si procede, adaptar la lista de objetivos y métricas a su contexto sectorial.  
3. **Validar y ajustar la matriz PRAGMATIC (`matriz-evaluacion…`)** en talleres con stakeholders.  
4. **Alinear con ENS/NIS2 (`mapeo-preguntas-gqm-normativa…`)** para asegurar trazabilidad regulatoria.  
5. **Construir el Excel (`plantilla-excel…`)** e implementar fórmulas y parámetros según la realidad económica de la organización.  
6. **Preparar el reporte ejecutivo (`plantilla-reporte-ejecutivo…`)** y adaptarlo a la plantilla corporativa de presentaciones.

---

## 3. Escenarios de aplicación

- **Organización individual** (ministerio, consejería, gran empresa):  
  - Diagnóstico interno de madurez SASE/Zero Trust.  
  - Justificación de roadmap y de inversiones.  

- **Estudio sectorial** (sanidad, educación, justicia, operadores críticos):  
  - Comparar niveles de madurez y brechas comunes.  
  - Proponer líneas de acción coordinadas (por ejemplo, desde un regulador sectorial).

- **Análisis nacional**:  
  - Medir la contribución de SASE al cumplimiento ENS/NIS2.  
  - Incluir indicadores SASE en informes de estado de la ciberseguridad nacional.

---

## 4. Personalización

El kit está concebido como una **plantilla avanzada**, no como dogma. Es recomendable:

- Ajustar objetivos GQM a estrategias propias (plan director, hoja de ruta Zero Trust, etc.).  
- Modificar la lista de métricas según la realidad tecnológica (distintos proveedores SASE, modelos de despliegue).  
- Revisar periódicamente ponderaciones y puntuaciones PRAGMATIC, especialmente si cambian ENS/NIS2 o las prioridades de negocio.

---

## 5. Buenas prácticas

- Mantener **versionado** del kit (v1, v2…) indicando cambios en métricas, pesos o parámetros.  
- Documentar hipótesis económicas empleadas en los cálculos de ROI.  
- Asegurar que ningún panel de indicadores se llena de métricas que no hayan pasado, como mínimo, una reflexión PRAGMATIC básica.

---

## 6. Cierre

Si este kit consigue que, en la próxima reunión de estado SASE, alguien pregunte “¿qué objetivo GQM soporta esta métrica?” antes de añadir otro gráfico, habrá cumplido con creces su cometido.
