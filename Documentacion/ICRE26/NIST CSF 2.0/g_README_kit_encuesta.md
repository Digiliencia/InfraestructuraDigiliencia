# README – Kit de Encuesta NIST CSF 2.0  
## Instrucciones rápidas para uso y adaptación

---

## 1. Contenido del kit

El kit incluye los siguientes archivos en formato Markdown:

- `a_modelo_encuesta.md` – Cuestionario completo para responsables de ciberseguridad.  
- `b_guia_metodologica.md` – Orientación sobre diseño, aplicación y análisis.  
- `c_narrativa_explicativa.md` – Texto explicativo para acompañar informes o comunicaciones.  
- `d_mapeo_normativo.md` – Tabla de mapeo de preguntas a marcos normativos y estándares.  
- `e_plantilla_igm_roi.md` – Especificación de una hoja Excel para cálculo de IGM y ROI.  
- `f_plantilla_reporte_ejecutivo.md` – Guion para preparar presentación ejecutiva (PowerPoint).  

---

## 2. Cómo empezar en 5 pasos

1. **Revisar y ajustar el modelo de encuesta** (`a_modelo_encuesta.md`).  
   - Adaptar lenguaje y ejemplos al sector o tipo de organización.  
   - Añadir o eliminar preguntas según alcance.

2. **Definir el modo de aplicación**.  
   - Formularios online (recomendado).  
   - Entrevistas guiadas (para análisis cualitativo más profundo).

3. **Construir el Excel de cálculo** a partir de `e_plantilla_igm_roi.md`.  
   - Crear las hojas `Respuestas`, `Madurez_CSFFunciones`, `IGM`, `ROI_Ciberseguridad`, `Dashboard`.  
   - Probar con datos de ejemplo.

4. **Planificar el análisis y el reporte**.  
   - Usar `b_guia_metodologica.md` para marcar el proceso de análisis.  
   - Utilizar `f_plantilla_reporte_ejecutivo.md` para el informe a la dirección.

5. **Comunicar resultados**.  
   - Incorporar fragmentos de `c_narrativa_explicativa.md` al informe o presentación.  
   - Acordar un ciclo de repetición (p.ej., anual) para seguimiento.

---

## 3. Personalización para distintos contextos

- **Administración pública**:  
  - Ajustar referencias a ENS, NIS2 y obligaciones sectoriales.  
  - Incorporar preguntas específicas de servicios públicos digitales.

- **Operadores esenciales y críticos**:  
  - Reforzar secciones de cadena de suministro y continuidad.  
  - Mapear con más detalle a NIS2, DORA u otras normas sectoriales.

- **Empresas privadas medianas/grandes**:  
  - Simplificar algunos apartados normativos, enfatizando ROI y alineamiento con negocio.  
  - Destacar indicadores orientados a reducción de pérdidas y reputación.

---

## 4. Buenas prácticas de uso

- Mantener un equilibrio entre sinceridad y diplomacia al interpretar resultados.  
- Evitar usar el IGM como arma arrojadiza; es un instrumento de mejora, no un boletín de notas.  
- Documentar cambios en el cuestionario a lo largo del tiempo para no distorsionar las comparaciones.  

---

## 5. Licencia y créditos

- Este kit está pensado para ser utilizado, adaptado y extendido en proyectos de evaluación de madurez en ciberseguridad alineados con NIST CSF 2.0.  
- Se recomienda mantener la referencia al marco NIST y a las normas asociadas cuando se redistribuya o incorpore en otros trabajos.
