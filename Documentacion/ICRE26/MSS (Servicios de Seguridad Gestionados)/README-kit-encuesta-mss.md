## ¿QUÉ ES ESTE KIT?

El **Kit de Encuesta MSS** es un instrumento de diagnóstico integral diseñado para evaluar la madurez de las organizaciones españolas en materia de Servicios de Seguridad Gestionados (MSS), con perspectiva comparativa europea y mundial.

Ha sido elaborado por un consorcio de expertos en ciberseguridad, criptografía, arquitectura Zero Trust, riesgo actuarial y resiliencia digital, sobre la base de las fuentes de más alta calidad disponibles a abril de 2026.

**¿Para quién es?**
- CISOs que necesitan un diagnóstico riguroso y presentable ante el Consejo
- Directores de Riesgos que necesitan cuantificar la exposición financiera al ciberriesgo
- Consultores que necesitan una herramienta estandarizada de evaluación de clientes
- Reguladores e investigadores interesados en benchmarking sectorial a escala nacional
- Equipos de cumplimiento que preparan la adecuación a NIS2, ENS, DORA o ISO 27001

**¿Cuándo usarlo?**
- Como diagnóstico inicial antes de iniciar un proyecto de mejora de ciberseguridad
- Antes de renegociar contratos con proveedores MSS/MSSP
- Como ejercicio preparatorio para una auditoría ENS, NIS2 o ISO 27001
- Como herramienta de benchmarking sectorial y comparativa con el mercado europeo
- Como instrumento de seguimiento anual para medir la evolución de la madurez

## MAPA DE USO SEGÚN PERFIL

### Si eres CISO o Responsable de Seguridad:

```
1. Lee la NARRATIVA EXPLICATIVA (c) para entender el marco conceptual
2. Distribuye la ENCUESTA INTEGRAL (a) a los perfiles relevantes de tu organización
3. Implementa la PLANTILLA EXCEL (e) para calcular el IGM-MSS y el ROSI
4. Prepara el REPORTE EJECUTIVO (f) para presentar al Consejo
5. Usa el MAPEO NORMATIVO (d) para identificar las brechas regulatorias prioritarias
```

### Si eres Consultor o Auditor externo:

```
1. Lee la GUÍA METODOLÓGICA (b) completa antes de aplicar el instrumento
2. Adapta la ENCUESTA INTEGRAL (a) a la modalidad más apropiada (autoaplicación/entrevista/taller)
3. Implementa la PLANTILLA EXCEL (e) como herramienta de análisis
4. Entrega el REPORTE EJECUTIVO (f) como salida del proyecto
5. Referencia el MAPEO NORMATIVO (d) para respaldar recomendaciones con base legal
```

### Si eres Director de Riesgos o CFO:

```
1. Lee el RESUMEN EJECUTIVO en la NARRATIVA EXPLICATIVA (c) — Secciones II y IV
2. Focaliza en la PLANTILLA EXCEL (e) — Hoja 4 (ROSI) y Hoja 5 (Benchmark)
3. Revisa el REPORTE EJECUTIVO (f) — Diapositivas 9 y 14 (ROSI y decisiones)
```

### Si eres responsable de Cumplimiento / Legal:

```
1. El MAPEO NORMATIVO (d) es tu documento de referencia principal
2. Complementa con las preguntas del Bloque 4 de la ENCUESTA INTEGRAL (a)
3. Identifica las brechas de NIS2, ENS, DORA y GDPR usando la tabla de consecuencias
```

## REQUISITOS TÉCNICOS MÍNIMOS

### Para la Encuesta (a):
- Cualquier editor de Markdown (VS Code, Typora, Obsidian, etc.)
- O conversión a formulario web (Google Forms, Microsoft Forms, Typeform, SurveyMonkey)
- O impresión directa del archivo .md renderizado como PDF

### Para la Plantilla Excel (e):
- **Microsoft Excel 365** o **Excel 2019+** (recomendado para compatibilidad completa con fórmulas BUSCARV, PROMEDIO.SI, gráfico de radar)
- **LibreOffice Calc 7.x** (compatible, puede requerir ajustes menores en gráficos avanzados)
- **Google Sheets** (compatible con funciones básicas; gráfico de radar puede requerir complemento)
- Memoria RAM mínima: 4GB para archivo completo con macros activas

### Para el Reporte Ejecutivo (f):
- **Microsoft PowerPoint 365** (recomendado)
- **Google Slides** (compatible con estructura propuesta)
- **Canva Pro** (alternativa de diseño avanzado)
- **LibreOffice Impress** (funcional con limitaciones de diseño)

### Para los archivos .md:
- Renderizado web: GitHub, GitLab, cualquier visor Markdown
- Conversión a PDF: Pandoc, Typora Export, VS Code + extensión Markdown PDF
- Conversión a Word: `pandoc archivo.md -o archivo.docx`

## ADVERTENCIA LEGAL

Este kit es un instrumento de diagnóstico y apoyo a la toma de decisiones. **No constituye asesoramiento legal ni garantía de cumplimiento normativo.** Para decisiones con consecuencias legales específicas, consulte siempre con asesores jurídicos especializados en derecho de la ciberseguridad y protección de datos. Los benchmarks incluidos son estimaciones derivadas de fuentes públicas y no reemplazan auditorías formales acreditadas (ENAC) ni certificaciones reconocidas (ISO 27001, ENS, EUMSS).
