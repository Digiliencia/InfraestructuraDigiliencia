# README – Kit de Encuesta HTMM

## 1. Contenido del kit

Este kit incluye los siguientes archivos en formato Markdown:

- `a_modelo_encuesta.md`: cuestionario completo para distribuir a las organizaciones.
- `b_guia_metodologica.md`: explicación del marco conceptual, diseño de la encuesta y recomendaciones de análisis.
- `c_narrativa_explicativa.md`: texto divulgativo para acompañar la encuesta en comunicaciones y presentaciones.
- `d_mapeo_normativo.md`: tabla de correspondencia entre preguntas y referencias normativas/estratégicas.
- `e_plantilla_excel_igm_roi.md`: instrucciones para configurar un libro de Excel con cálculo de IGM y ROI.
- `f_plantilla_reporte_ppt.md`: esquema de presentación ejecutiva en formato PowerPoint.

## 2. Uso recomendado

1. **Adaptación mínima y contextualización**  
   Revise los textos para adaptarlos a su realidad sectorial y geográfica (por ejemplo, referencias específicas a España, reguladores nacionales, etc.).

2. **Implementación de la encuesta**  
   Transforme `a_modelo_encuesta.md` en el formato de la herramienta de encuesta que utilice (formularios web, sistema interno, etc.). Mantenga la lógica y el orden de las preguntas para facilitar el análisis comparativo.

3. **Comunicación a participantes**  
   Utilice `c_narrativa_explicativa.md` para acompañar las invitaciones y contextualizar la relevancia del estudio. Un toque de humor suele incrementar la tasa de respuesta.

4. **Análisis de resultados**  
   Utilice `b_guia_metodologica.md` y `e_plantilla_excel_igm_roi.md` para estructurar el análisis, construir indicadores y preparar visualizaciones.

5. **Reporte a la dirección y partes interesadas**  
   Apóyese en `f_plantilla_reporte_ppt.md` para construir un informe ejecutivo que traduzca los resultados en mensajes inteligibles para comités de dirección y órganos de gobernanza.

## 3. Referencias conceptuales

El kit se inspira principalmente en:

- El Hybrid Threat Modeling Method (hTMM) del Software Engineering Institute, que combina Security Cards, PnG, STRIDE y SQUARE para mejorar la cobertura y consistencia de los modelos de amenaza. [web:12][web:6][web:5]
- El informe ENISA Threat Landscape 2025, que describe un entorno de amenazas cada vez más híbrido y profesionalizado a nivel de la Unión Europea. [web:13][web:7][web:18]
- Trabajos recientes que integran marcos como STRIDE, MITRE ATT&CK y CVSS con el modelado de amenazas para cuantificar riesgos de forma más rigurosa. [web:8][web:15]

## 4. Consideraciones éticas y de confidencialidad

- Defina claramente a los participantes el uso previsto de los datos, las medidas de anonimización y los posibles usos agregados para informes sectoriales o nacionales.
- Evite publicar resultados que permitan identificar organizaciones concretas sin su consentimiento explícito.
- Tenga en cuenta las obligaciones legales aplicables en materia de protección de datos y secretos empresariales.

## 5. Evolución del kit

La práctica del modelado de amenazas y el propio paisaje de amenazas evolucionan con agilidad, por decirlo suavemente. Se recomienda revisar y actualizar periódicamente:

- Las referencias normativas (por ejemplo, cambios en NIS2, guías de ENISA u otras autoridades).
- La taxonomía de amenazas y actores, incorporando nuevas tendencias (uso de IA generativa por atacantes, cadenas de suministro de software, etc.).
- Las ponderaciones y modelos de ROI, para ajustarlos a datos empíricos recientes.

Si tras un tiempo este kit le parece obvio, limitado o ingenuo, considéralo un éxito: significará que el nivel medio de madurez en modelado de amenazas ha subido y que ya está listo para la siguiente iteración.
