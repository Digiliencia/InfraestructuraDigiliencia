# README – Kit GQM + PRAGMATIC para indicadores HTMM

## 1. Qué incluye este kit

Este kit complementa la encuesta HTMM con un marco explícito para conectar objetivos estratégicos, preguntas operativas y métricas técnicas, y para evaluar la calidad de estas métricas.

Archivos incluidos:

- `a_marco_GQM_PRAGMATIC.md`: descripción del marco integrativo GQM + PRAGMATIC aplicado a indicadores de modelado de amenazas inspirados en hTMM y ENISA TL 2025. [web:6][web:16][web:13]
- `b_matriz_PRAGMATIC.md`: matriz de evaluación PRAGMATIC de un conjunto de métricas representativas.
- `c_narrativa_explicativa_GQM_PRAGMATIC.md`: texto explicativo en prosa sobre el sentido y uso del marco.
- `d_mapeo_normativo_GQM.md`: tabla que vincula objetivos, preguntas y métricas con requisitos normativos y estratégicos.
- `e_plantilla_excel_IGM_ROI_GQM.md`: instrucciones para implementar en Excel el cálculo de un IGM basado en métricas GQM ponderadas por PRAGMATIC, y un modelo de ROI. [web:32][web:34]
- `f_plantilla_ppt_GQM_PRAGMATIC.md`: guión para elaborar un reporte ejecutivo en PowerPoint.

## 2. Cómo usarlo junto con la encuesta HTMM

1. **Definir el alcance**  
   Decida si el marco se aplicará a nivel nacional, sectorial o organizativo, y qué subconjunto de métricas es realmente manejable en su contexto.

2. **Configurar la recogida de datos**  
   Adapte la encuesta HTMM (y otros mecanismos de reporting) para garantizar que las preguntas permiten calcular las métricas M1.1–M5.3 y cualesquiera otras que decida incorporar.

3. **Implementar la plantilla Excel**  
   Utilice `e_plantilla_excel_IGM_ROI_GQM.md` para construir el libro de cálculo, incorporando la matriz PRAGMATIC como base para los pesos.

4. **Calibrar PRAGMATIC**  
   Revise `b_matriz_PRAGMATIC.md` con un grupo de expertos y ajuste las puntuaciones de cada métrica para reflejar prioridades locales (por ejemplo, mayor peso a alineación con ENISA, o a trazabilidad hacia riesgos).

5. **Construir el relato**  
   Apóyese en `c_narrativa_explicativa_GQM_PRAGMATIC.md` y `f_plantilla_ppt_GQM_PRAGMATIC.md` para explicar el marco a decisores y partes interesadas, evitando que se perciba como una mera acumulación de números.

## 3. Actualización y mantenimiento

El marco debe revisarse periódicamente para:

- Incorporar nuevas categorías de amenazas y tendencias descritas por ENISA y otras fuentes. [web:13][web:7]
- Ajustar los objetivos (G) y métricas (M) a cambios regulatorios (p.ej. evolución de NIS2, estrategias nacionales de ciberseguridad).
- Depurar métricas que se demuestren poco útiles o difíciles de obtener, y reforzar aquellas que aporten mayor valor predictivo y accionable.

## 4. Licencia de uso intelectual (no jurídica)

Si al cabo de un tiempo este marco le parece obvio, limitado o ingenuo, considérelo un éxito parcial: significará que el nivel de discusión sobre indicadores de modelado de amenazas ha madurado y que ya está listo para una versión 2.0.

Mientras tanto, úselo como caja de herramientas y no como dogma.
