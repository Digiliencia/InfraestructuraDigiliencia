# (g) README del Kit de Encuesta APT

## 1. Contenido del kit

El kit incluye los siguientes archivos:

- **(a) encuesta_APT_modelo.md**: modelo de encuesta integral en formato Markdown.
- **(b) guia_metodologica_APT.md**: guía metodológica detallada para diseño muestral, escalas y cálculo de índices.
- **(c) narrativa_explicativa_APT.md**: narrativa explicativa para acompañar la encuesta en comunicaciones internas o estudios.
- **(d) mapeo_preguntas_normativa_APT.md**: tabla de mapeo de cada pregunta a requisitos normativos y marcos de referencia.
- **(e) plantilla_excel_IGM_ROI_APT.md**: instrucciones para implementar en Excel el cálculo de IGM y ROI.
- **(f) plantilla_reporte_ejecutivo_PPT_APT.md**: guía de estructura de presentación ejecutiva.
- **(g) README_kit_encuesta_APT.md**: este documento.

## 2. Uso recomendado

1. Adaptar el **modelo de encuesta** al contexto sectorial y terminología propia de la organización.
2. Implementar la **plantilla de Excel** para captura de respuestas y cálculo de indicadores.
3. Utilizar la **guía metodológica** para definir muestra, frecuencia de aplicación y análisis estadístico.
4. Construir, a partir de la plantilla, un **reporte ejecutivo** para la alta dirección.
5. Mantener la **narrativa explicativa** como documento de acompañamiento para campañas de sensibilización y para contextualizar los resultados.

## 3. Licencia y créditos

- El kit se ofrece como base para proyectos de evaluación de madurez frente a APT. Puede ser adaptado, ampliado o reducido según las necesidades de cada organización.
- Se recomienda conservar las referencias conceptuales a marcos reconocidos (NIS2, NIST CSF, ISO 27001, MITRE ATT&CK, etc.) para facilitar el alineamiento con normativas y buenas prácticas.

## 4. Personalización avanzada

- Añadir módulos específicos por sector (sanidad, financiero, industrial, administración pública) introduciendo nuevas preguntas en los bloques ya definidos.
- Incluir referencias adicionales a marcos regulatorios sectoriales (DORA, NERC‑CIP, PCI DSS, ENS por categoría, etc.).
- Integrar los resultados de la encuesta con herramientas de BI para análisis avanzado.

## 5. Próximos pasos sugeridos

- Definir una periodicidad (por ejemplo, anual) para replicar la encuesta y comparar resultados.
- Establecer objetivos de mejora (por bloque) y monitorizar la evolución del IGM.
- Incorporar los resultados en el cuadro de mando de ciberseguridad de la organización.

El kit nace para ser usado, versionado y discutido. Si genera conversación (y algún que otro presupuesto aprobado), habrá cumplido dignamente su misión.
