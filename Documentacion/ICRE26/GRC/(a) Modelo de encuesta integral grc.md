## 4. Capacidades técnicas y Zero Trust

### 4.1. Identidad y acceso

**P4.1.1. Nivel de madurez en gestión de identidades (IAM) y accesos privilegiados (PAM)**

- [ ] Gestión manual / usuarios locales sin control centralizado  
- [ ] Directorio centralizado básico (AD u otros)  
- [ ] IAM con control de roles y grupos, revisiones periódicas de accesos  
- [ ] IAM avanzado con SSO, MFA generalizado, PAM para accesos privilegiados  
- [ ] Modelo Zero Trust basado en identidad (acceso condicional, principios de mínimo privilegio “de verdad”)

**P4.1.2. Cobertura de autenticación multifactor (MFA)**

- [ ] No se utiliza MFA  
- [ ] Limitado a accesos VPN / remotos  
- [ ] Extendido a administradores y usuarios sensibles  
- [ ] Aplicado a todos los usuarios internos  
- [ ] Aplicado a todos los usuarios internos y externos relevantes, incluyendo clientes y proveedores

### 4.2. Red, segmentación y acceso remoto

**P4.2.1. Enfoque actual de arquitectura de red**

- [ ] Red plana con controles perimetrales básicos  
- [ ] Segmentación básica entre entornos (oficina / producción / DMZ)  
- [ ] Segmentación avanzada con microsegmentación en entornos críticos  
- [ ] Enfoque Zero Trust Network Access (ZTNA) en sustitución o refuerzo de VPN  
- [ ] Arquitectura predominantemente cloud con principios Zero Trust integrados

**P4.2.2. Uso de soluciones ZTNA / SSE / SASE**

- [ ] Ninguna  
- [ ] PoC / piloto  
- [ ] Implantación parcial en algunos colectivos  
- [ ] Implantación extendida a la mayoría de usuarios externos / remotos  
- [ ] Modelo predominante de acceso a recursos internos

### 4.3. Datos, detección y respuesta

**P4.3.1. Protección de datos sensibles (DLP / cifrado / clasificación)**

- [ ] No existe clasificación formal de la información  
- [ ] Clasificación básica, sin controles técnicos asociados  
- [ ] Clasificación con controles de cifrado en tránsito y en reposo  
- [ ] DLP implementado en correo y web para datos críticos  
- [ ] Clasificación, cifrado y DLP integrados, con gobierno de datos y auditoría

**P4.3.2. Capacidades de detección y respuesta (SOC, SIEM, EDR/XDR)**

- [ ] Sin capacidades dedicadas (solo antivirus / firewall básico)  
- [ ] Monitorización parcial mediante registros y herramientas puntuales
