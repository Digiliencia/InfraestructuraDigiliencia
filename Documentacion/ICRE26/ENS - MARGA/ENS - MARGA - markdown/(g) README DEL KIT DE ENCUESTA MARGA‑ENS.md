# (g) README DEL KIT DE ENCUESTA MARGA‑ENS

## 1. Descripción general

El **Kit de Encuesta MARGA‑ENS** es un conjunto de recursos para diseñar, administrar y explotar una encuesta sobre indicadores y métricas de ciberseguridad y ciberresiliencia en organizaciones alineadas con el Esquema Nacional de Seguridad (ENS) y el modelo IMC de INCIBE.[web:26][web:27][web:35]

Su propósito es ayudar a:

- Diagnosticar el grado de uso de indicadores alineados con ENS e IMC.  
- Construir un Índice Global de Madurez (IGM) por organización/sector.  
- Explorar relaciones entre madurez, riesgo y retorno de inversión (ROI).  
- Ofrecer insumos para decisiones estratégicas y políticas públicas.

---

## 2. Contenido del kit

El kit está formado por los siguientes archivos Markdown (convertibles a los formatos que se deseen):

1. `a_modelo_encuesta.md` – Modelo de encuesta integral para responsables de seguridad.  
2. `b_guia_metodologica.md` – Guía para diseño, administración y análisis de la encuesta.  
3. `c_narrativa_explicativa.md` – Texto explicativo para contextualizar la encuesta ante partes interesadas.  
4. `d_mapeo_preguntas_normativa.md` – Tabla que vincula cada pregunta con ENS, IMC y marcos relacionados.  
5. `e_plantilla_excel_igm_roi.md` – Descripción de la estructura de Excel para cálculo de IGM y ROI.  
6. `f_plantilla_reporte_ejecutivo.md` – Guión de presentación ejecutiva tipo PowerPoint.  
7. `g_README.md` – Este documento.

## 3. Requisitos previos

- Conocimiento básico de ENS y su proceso de adecuación (categorización, DoA, etc.).[web:26][web:19]  
- Familiaridad con el Diccionario IMC de INCIBE (recomendable, aunque no imprescindible).[web:27][web:35]  
- Herramientas ofimáticas estándar: editor de texto, Excel (o equivalente), y software de presentaciones.

## 4. Cómo utilizar el kit

### 4.1. Paso 1 — Adaptar la encuesta

- Partir de `a_modelo_encuesta.md`.  
- Revisar cada bloque y adaptar el lenguaje a la organización o al sector, sin alterar en exceso la estructura, para preservar comparabilidad.  
- Traducir a la herramienta de encuestas elegida (formulario web, sistema interno, etc.).

### 4.2. Paso 2 — Preparar la codificación y el Excel

- Utilizar `e_plantilla_excel_igm_roi.md` para crear el libro de Excel correspondiente.  
- Definir en la hoja `Codificacion` los códigos numéricos acordes con las opciones de respuesta.  
- Asegurar que los identificadores de preguntas (P5–P57, etc.) se mantienen coherentes.

### 4.3. Paso 3 — Administrar la encuesta

- Consultar `b_guia_metodologica.md` para recomendaciones sobre destinatarios, confidencialidad y tiempos.  
- Enviar la encuesta a las organizaciones seleccionadas y establecer un plazo razonable de respuesta.  
- Supervisar la calidad de las respuestas y resolver dudas de interpretación.

### 4.4. Paso 4 — Analizar y construir el IGM

- Importar las respuestas en la hoja `Datos_Encuesta`.  
- Calcular subíndices e IGM según las fórmulas definidas.  
- Explorar variantes de ponderación si es necesario para contextos específicos.

### 4.5. Paso 5 — Preparar el reporte ejecutivo

- Seguir `f_plantilla_reporte_ejecutivo.md` como guión para elaborar la presentación de resultados.  
- Ajustar mensaje y gráficos al público objetivo (técnico, directivo, regulador).

### 4.6. Paso 6 — Comunicar y planificar acciones

- Utilizar `c_narrativa_explicativa.md` para contextualizar la encuesta y evitar interpretaciones defensivas.  
- Presentar resultados, discutir prioridades y definir hojas de ruta de mejora.

## 5. Actualización del kit

El entorno normativo y técnico evoluciona: ENS se actualiza, IMC se amplía, la UE refuerza la regulación a través de NIS2, DORA y otras iniciativas.[web:26][web:27][web:7][web:12] Por ello:

- Se recomienda revisar el kit al menos cada dos años.  
- Las preguntas pueden ajustarse para reflejar nuevas obligaciones o buenas prácticas (p. ej. IA, 5G, cadena de suministro).  
- Cualquier cambio debe documentarse para preservar trazabilidad entre versiones.
